📖 如何写一篇新文章

第 1 步：创建文章文件

在 C:\Users\35359\my-blog\_posts\ 目录下新建文件，文件名必须严格按这个格式：

格式：YYYY-MM-DD-文章标题.md
示例：2026-07-06-今天学了啥.md

第 2 步：写文章内容

文件开头必须包含下面这段 Front Matter（分隔线不能省）：

---
layout: post
title: "你的文章标题"
date: 2026-07-06 10:00:00 +0800
tags: [标签1, 标签2]
---

正文从这里开始，支持 Markdown 语法。

## 二级标题

可以放 **加粗**、*斜体*、`代码`、[链接](网址)

- 列表项 1
- 列表项 2

> 引用一段话

也可以插入图片：![描述](图片链接)
💡 小技巧：文件开头那段 --- 之间的内容叫 Front Matter，告诉 Jekyll 这篇文章的标题、日期和标签。下面是正文，随便写。

第 3 步：推送到 GitHub

打开 PowerShell，粘贴以下命令：

cd C:\Users\35359\my-blog
git add _posts/你的文件名.md
git commit -m "发布：文章标题"
git push
⚠️ 注意：推送前确保梯子开着！git 已经配置了走 127.0.0.1:7890 代理。

第 4 步：查看结果

推送后等 1-2 分钟，打开 https://fyl34222220.github.io 刷新即可看到新文章。

🏷️ 标签 & 分类

在文章的 Front Matter 里可以添加标签：

tags: [编程, Python, 教程]
categories: 技术

🗑️ 删除文章

删除本地 _posts 下对应的文件，然后推送：

cd C:\Users\35359\my-blog
git rm _posts/要删的文章.md
git commit -m "删除文章"
git push

⚡ 常用命令速查

# 进入博客目录
cd C:\Users\35359\my-blog

# 查看有哪些文件改了
git status

# 添加所有改动
git add -A

# 提交
git commit -m "描述你做了什么"

# 推送
git push

# 一次搞定（改完所有文章后）
git add -A && git commit -m "更新" && git push
